2025 年 LLM 大事记：Agent 元年正式到来

背景2025 年是公认的"Agent 元年"。大模型能力从"单轮问答"飞跃到"自主完成多步骤任务"，这一跃迁直接影响你未来做的每一个 AI 产品。

核心进展强化学习（RLVR）成为 2025 年最大的技术突破——模型通过强化学习自发学会"推理"，表现为 DeepSeek R1、OpenAI o3 等推理模型的爆发。能力提升不靠堆模型大小，靠"思考时间"的延长。

Agent 里程碑Claude Code 被视为"第一个真正意义上的 LLM Agent"，它以循环方式将工具调用与推理串联起来，解决复杂问题。MCP（模型上下文协议）在 2025 年爆炸式普及，成为工具调用的事实标准。

市场格局Anthropic 以 32% 份额成为企业 AI 市场第一，超越 OpenAI（25%）。企业级 Agent 中最热门的场景是客服（这正是你的主战场）和内部效率工具。

来源：Simon Willison's Weblog / Karpathy 年度回顾 / Menlo Ventures 市场报告 · 阅读原文 →
🔑 你需要记住的 3 个关键点
Agent ≠ 聊天机器人：Agent 能循环调用工具、自主规划步骤，你过去做的坐席辅助（意图识别→知识推荐→工单填写）其实已经是早期 Agent 形态，面试时要这样表达。
MCP 是新基础设施：就像当年 REST API 统一了接口，MCP 正在统一 AI 工具调用方式。你的下一个产品方案里，可以考虑如何利用 MCP 打通行内系统。
质量仍是最大障碍：调研显示 32% 的团队把"模型输出质量"列为生产部署第一障碍。你在金融场景做 Badcase 分析、A/B Test 的经验，是稀缺的差异化竞争力。
💭 今日思考题
你的坐席辅助项目（意图识别 → RAG 话术推荐 → 工单 NER）本质上是一个 3 步 Agent。如果今天用 MCP 重新设计这个系统，你会怎么拆解工具调用？

Notion 3.0：从协作工具到 AI 知识工作 Agent

产品演进Notion 于 2025 年 9 月发布 3.0 版本，把 AI 从"功能"升级为"Agent"——核心差异是：AI 可以代替用户在 Notion 里完成所有操作，包括创建文档、搭建数据库、跨工具搜索，支持连续运行超 20 分钟。

关键设计决策Notion AI Agent 的独特之处在于"记忆库"机制：Agent 能整合用户 Notion 里所有可访问的信息（包括 Slack、Google Drive、GitHub），上下文不依赖单次对话，而是持久化在知识库里——这解决了企业 AI 最大的痛点：AI 不了解公司背景。

商业逻辑Notion AI 上线 2 个月 ARR 新增超 1000 万美元，4 个月用户突破 400 万。AI 不是独立产品，而是把用户锁在 Notion 生态的"护城河"——这是"AI 是功能而非产品"的最佳诠释。

来源：Notion 官方发布 / 少数派分析 · 阅读原文 →
🔑 金融 PM 的借鉴视角
Notion 的"记忆库" = 你的"客户画像"：你做过的智能营销系统（客群筛选+旅程经营），本质和 Notion Agent 的记忆机制一样——让 AI 在长周期内保持对用户的理解。这是可迁移的产品设计经验。
AI 应该嵌入工作流，而不是独立窗口：Notion AI 不要求用户跳到另一个 Tab，而是在原有场景内触发——你的坐席辅助也是这个逻辑，但可以更进一步思考"主动推送"和"被动等待"的边界。
MCP 生态构建是产品壁垒：Notion 接入了 Lovable、Perplexity、HubSpot 等 MCP 合作伙伴，让 AI 能跨系统完成任务。银行的 AI 产品也可以思考：如何通过 MCP 打通行内各系统，让 Agent 真正端到端完成业务。
💭 今日思考题
Notion Agent 靠"记忆库"让 AI 了解公司上下文。你的银行智能客服系统里，有没有类似的"持久上下文"机制？如果没有，这是一个值得做的方向吗？有哪些合规风险？
预计耗时 30 分钟

把你的项目经历翻译成 AI 原生语言

你的简历已经很好，但还可以用 2025 年的 AI 语言体系重新表达，让 AI 原生公司的面试官一眼看出你懂大模型时代的产品逻辑。

对照练习以下是一个"旧表达 → 新表达"的示例框架：

❌ 旧表达："基于 BERT/NLP 定义了 14 个业务意图分类，准确率从 35% 提升至 85%"

✅ 新表达："设计了一个 3 层 Intent Agent：（1）流式 ASR 实时转写；（2）微调 BERT 做 14 分类意图路由；（3）低置信度时触发人机协同兜底。通过 Badcase 驱动的数据飞轮，将意图准确率从 35% → 85%，并建立了持续迭代的评估体系。"

关键差异新表达体现了：架构思维（3 层设计）、数据闭环意识（数据飞轮）、Agent 范式理解（工具调用+兜底策略）。这些是 AI 原生 PM 最看重的思维方式。

🔑 你的 3 个可重新表达的项目亮点
坐席辅助中的 RAG：你搭建的"话术地图"推荐 = 早期 RAG 系统。可以表达为："设计了基于向量数据库的 Retrieval-Augmented Generation 系统，根据实时对话上下文动态检索最优应答策略，覆盖 XX 个业务场景。"
RPA+AI 项目：这是 Agentic Workflow 的早期实践。可以表达为："主导设计 RPA+LLM 混合 Agent，以 OCR+NLP 作为感知层，规则引擎作为决策层，RPA 作为执行层，构建端到端自动化工作流。"
国产化迁移中的配置平台化：你把 IVR/话术/路由抽象为可视化配置中台，这是"AI 策略快速实验底座"，可以表达为："构建了 AI 策略配置平台，支持千人千面路由策略的动态热更新和灰度实验，降低算法迭代的工程耦合度。"
